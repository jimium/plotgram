# Drawify 项目愿景与定位

## 一句话定位

> **Drawify 是面向 AI Agent 的原生图表描述语言与渲染引擎。**

Drawify 不是 Mermaid 的替代品，而是一种为 AI 生成场景从零设计的图表语言。它解决的核心问题是：当 LLM 需要生成图表时，现有工具的语法对机器不友好。

---

## 背景与问题

### 现有工具的根源假设

Mermaid、PlantUML、Graphviz DOT 都是在 AI 时代之前设计的工具。它们的语法为**人类手写**优化：

- **语法变体多** — 同一种图可以有多种写法，人类觉得灵活，AI 觉得困惑
- **隐式规则** — 箭头样式、布局方向、子图嵌套等规则分散，需要大量隐性知识
- **错误静默** — 语法错误时渲染空白，不告诉 Agent 哪里错了、怎么修

### AI 生成图表的现状

| 指标 | 现状 |
|------|------|
| LLM 生成 Mermaid 正确率 | 约 70-80%，复杂图更低 |
| 错误反馈 | 渲染失败返回空白，无结构化错误信息 |
| 迭代修复能力 | 文本 Diff 无意义，无法语义级修改 |
| 布局控制 | Agent 需要同时表达语义和布局，增加出错概率 |

### 根本矛盾

现有图表语言的设计假设是**"人类书写，机器渲染"**。
AI 时代的需求是**"机器书写，机器渲染，人类阅读"**。

这两者的设计目标完全不同。

---

## Drawify 的核心理念

### 1. 第一用户是 AI Agent

语法设计优先考虑 LLM 的生成模式：
- 有限的、明确的语法构造（减少幻觉空间）
- 强类型属性系统（减少歧义）
- 每个错误都有明确的错误码、位置、修复建议

### 2. 语义优先，布局后置

人类和 AI 画图的思维过程都是：**先有关系，后有布局**。

Drawify 只要求 Agent 表达"实体"和"关系"，布局完全由引擎智能推断。Agent 不需要（也不应该）关心节点坐标、连线弯曲方式等视觉细节。

### 3. 结构化可编程

Drawify 的 AST 是一等公民：
- 可以序列化为 JSON，供程序消费
- 支持语义级别的 Diff 和 Patch
- 可以被另一个 Agent 读取、理解、修改

## 目标用户

### 直接用户（API 调用者）

| 用户 | 使用方式 |
|------|----------|
| AI Agent | 生成 Drawify 文本 → 调用渲染 API → 展示给用户 |
| LLM 应用开发者 | 在 Prompt 中要求输出 Drawify → 嵌入渲染组件 |
| IDE / 编辑器插件 | 实时预览 .dfy 文件 |

### 间接用户（图表消费者）

| 用户 | 使用方式 |
|------|----------|
| 技术文档读者 | 阅读 Agent 生成的架构图、流程图 |
| 产品经理 | 阅读 AI 生成的用户旅程图、状态机图 |
| 开发者 | 阅读 AI 生成的系统架构、数据流图 |

---

## 产品边界

### Drawify 做什么

- 定义一套简洁、无歧义的图表描述语法
- 提供可靠的解析器（文本 → AST）
- 提供多格式渲染器（AST → SVG / JSON / PNG）
- 提供结构化错误反馈（供 Agent 自我修正）
- 提供 AST 的语义 Diff 能力

### Drawify 不做什么

- 不实现交互式图表编辑器（那是前端的事）
- 不实现 LLM 本身（Drawify 是 LLM 的输出格式）
- 不强制绑定特定前端框架（提供 SVG/JSON，任意框架可消费）
- 不支持手绘风格（那是 Excalidraw 的定位）

---

## 成功标准

### 短期（MVP 验证）

- LLM 生成 Drawify 的正确率 > 95%（对比 Mermaid 的 70-80%）
- 语法错误时，Agent 可以根据错误反馈在 1 次重试内修复

### 中期（产品可用）

- 被至少 2 个 LLM 应用集成（如 Cursor、ChatGPT 插件等）
- 支持 5 种以上图表类型（流程图、时序图、架构图、ER图、状态机）
- WASM 包体积 < 500KB，首屏渲染 < 100ms

### 长期（生态建设）

- 成为 AI Agent 生成图表的事实标准格式
- 建立 Drawify 的 schema 注册表（自定义图表类型扩展）
- 开源社区活跃贡献者 > 50

## 技术路线概览

```
drawify-core（共享核心）
├── parser    — 文本 → AST
├── validation — AST 语义验证
└── renderer  — AST → SVG/JSON/PNG

drawify-cli    — 命令行工具（本地文件处理）
drawify-server — Web API（远程渲染服务）
drawify-wasm   — WASM 包（浏览器内渲染）
```
