Mindmap Radial 径向模式

Mindmap 布局的默认模式，根节点居中辐射展开，一级分支左右交替排列，适合思维导图与知识树

BFS 左右交替辐射 Mindmap 模式 确定性 ≤ 200 节点

◎ 模式概述

Radial 是 Mindmap 布局的一种模式，而非独立算法。在 mindmap.rs 中， Mindmap 布局支持三种展开方向：radial（径向，默认）、top-to-bottom（自上而下）、 left-to-right（自左向右）。本页专门描述 Radial 模式的实现细节。

Radial 模式以根节点为中心，一级子节点左右交替辐射展开——偶数序号分支向右，奇数序号分支向左。 每个子树递归布局，y 方向按子树高度累进，x 方向按深度等间距。 节点尺寸根据角色差异化：根节点最大，主干节点中等，分支/叶节点较小。

该模式是确定性的——相同输入始终产生相同输出，适合 ≤ 200 个节点的图。

⇨ 左右交替辐射策略

Radial 模式的核心策略是左右交替辐射：一级子节点按声明顺序，偶数序号（0, 2, 4…）分支向右展开，奇数序号（1, 3, 5…）分支向左展开。 每个子树在各自方向上递归布局，y 方向按子树高度累进，x 方向按深度等间距递增。

分配过程

  1. 识别根节点（类型为 root/initial/start 的节点，或按度数评分选择）
  2. 一级子节点按声明顺序，偶数序号分支向右，奇数序号分支向左
  3. 每个分支递归布局子树：y 方向按子树高度累进，x 方向按深度等间距
  4. 根节点居中，左右两侧的子树分别向上/下累进排列

★ 根节点选择

根节点的选择直接影响思维导图的整体结构。算法采用两阶段策略：

阶段一：类型匹配

遍历连通分量中的所有节点，若某节点的 type 属性为 "root"、"initial" 或 "start"，则直接选为根节点。这允许用户通过声明式标注指定根。

阶段二：评分排序

若无类型匹配，则对每个节点计算四元组评分，选取最高分者作为根：

1
-indegree — 入度取负
入度越低越好，根节点通常不应有入边
2
outdegree — 出度
出度越高越好，根应连接尽可能多的子节点
3
total_degree — 总度数
总连接数越高，越适合作为图的中心
4
-entity_order — 声明顺序取负
同分时优先选择声明更早的节点，保证确定性
fn root_score(node_id, component_set, graph) -> (i32, i32, i32, i32) {
    (-indegree, outdegree, total_degree, -entity_order)
}

◰ 分支布局策略

每个一级分支独立递归布局子树，涉及子树高度计算和坐标分配两个核心步骤。

1. 子树高度计算

递归计算每个子树的总高度：叶节点高度 = 自身高度；内部节点高度 = 所有子树高度之和 + 子树间 V_GAP 间距。 子树高度决定了同方向相邻分支的 y 坐标累进量。

2. 坐标分配

对每个分支递归分配坐标：

方向坐标计算说明
x（水平） root_x ± (depth × LEVEL_GAP + CENTER_GAP) 右侧分支 +，左侧分支 -，深度递增
y（垂直） parent_y + subtree_offset 按子树高度累进，同方向分支依次排列

▦ 多连通分量支持

当图包含多个不连通的分量时，算法对每个分量独立执行布局，然后将各分量水平排列打包：

  1. 通过父子关系检测所有连通分量
  2. 每个分量独立选择根节点并布局
  3. 各分量归一化坐标后，水平排列打包

这保证了即使图不连通，布局仍然清晰有序，各分量互不干扰。

⚙ 关键参数

以下常量来自 mindmap.rs，Radial 模式与其他模式共享

参数默认值说明
PADDING48px画布边距
V_GAP22px同方向相邻分支的垂直间距
H_GAP70px父子节点之间的水平间距
LEVEL_GAP200px层级之间的水平间距
CENTER_GAP100px根节点与一级分支之间的间距

✓ 适用场景

  • 思维导图 — 中心主题 + 左右辐射展开，最经典的导图形态
  • 知识树 — 从核心概念出发，分支向外展开
  • 会议笔记 — 中心议题 + 分支论点
  • 树状结构 — 天然具有根和层次的结构

不适用场景

  • 节点数超过 200 的大图（分支过密导致可读性下降）
  • 非树形结构（网状图、有环图）——应考虑 Force-Directed 或 Circular
  • 需要强调垂直/水平层次的方向图——可使用 Mindmap 的 top-to-bottom / left-to-right 模式

▶ 算法流程

MindmapLayout::compute(diagram)  [mode = Radial]
  ├─ build_children_map()         // 构建父子关系映射
  ├─ find_root_id()               // 选择根节点
  │   ├─ 类型匹配: type=root/initial/start
  │   └─ 评分排序: (-indeg, outdeg, total_deg, -order)
  │
  ├─ layout_mode() = Radial       // 解析 layout_direction 属性
  │
  ├─ layout_radial(root, children, sizes)
  │   ├─ 按声明顺序遍历一级子节点
  │   │   ├─ 偶数序号 → 右侧分支
  │   │   └─ 奇数序号 → 左侧分支
  │   │
  │   ├─ 对每个分支递归布局子树
  │   │   ├─ compute_subtree_height()  // 计算子树高度
  │   │   └─ layout_subtree()          // 递归布局
  │   │       ├─ y: 按子树高度累进
  │   │       └─ x: 按深度 × LEVEL_GAP
  │   │
  │   ├─ 根节点居中 (CENTER_GAP)
  │   └─ 合并左右两侧子树坐标
  │
  ├─ build_node_sizes()           // 根据角色差异化尺寸
  │   ├─ root: 最大尺寸
  │   ├─ trunk: 中等尺寸
  │   └─ branch/leaf: 较小尺寸
  │
  ├─ compute_group_bounds()       // 计算分组边界
  └─ bounds_from_layout()         // 计算总尺寸
