边路由算法集

Drawify 提供了 5 种边路由算法，从最简单的直线连接到高级样条避障路径。 每种算法针对不同的可视化场景优化，与布局算法搭配使用以获得最佳效果。

5 Algorithms Orthogonal Default Obstacle Avoidance Parallel Edge Handling

📌 算法概览

边路由（Edge Routing）决定了图中节点之间连线的形状与走向。 好的边路由能减少交叉、避免遮挡、提升可读性。Drawify 的边路由算法体系分为三个层次：

基础层

Straight、Bezier — 简洁高效，适合边数较少的场景

结构层

Orthogonal — 规整折线，适合工程/架构图（渲染层自动圆角）

高级层

Circular、Spline — 环形布局适配 / 避障样条

🧩 五种边路由算法

Straight

edge_routing.rs

核心思想：源端口到目标端口的直线连接。平行边通过偏移量（offset）避免重叠。

低复杂度 无避障 平行边偏移

适用场景：简单关系图、ER 图、边数较少的图

Orthogonal

edge_routing_orthogonal/

核心思想：创建直角折线路径。使用通道/槽位（channel/slot）系统寻找可用的水平/垂直走廊，基于评分选择路径（惩罚弯折、交叉、长绕行），最后简化路径。

中等复杂度 推荐默认 避障评分 路径简化

适用场景：流程图、架构图（搭配 Sugiyama 布局的默认推荐）

Bezier

edge_routing_bezier.rs

核心思想：端口感知的自适应三次贝塞尔曲线。控制点沿端口方向延伸，并根据节点间距与转角关系自动调整；可通过 edge_routing: bezier { tension: … } 调节弧度。

低-中复杂度 平滑曲线 可调曲率

适用场景：思维导图、轻量可视化图、需要柔和曲线的通用图

Circular

edge_routing_circular.rs

核心思想：弧形边，专为环形布局设计。边沿圆弧路径行进，与圆形节点排列自然契合。

中等复杂度 环形布局专用 弧形路径

适用场景：状态图、循环关系图（搭配 circular 节点布局）

Spline

edge_routing_spline.rs

核心思想：可见性图避障 + Catmull-Rom 多段样条拟合。比 Bezier 更平滑，能绕开中间节点。

高复杂度 障碍避让 多段样条 C1 连续

适用场景：节点密集、需要避障的高质量路径；复杂图可视化

🔧 公共基础设施

边路由算法共享一组底层工具模块，负责几何计算、标签定位和平行边处理。

visibility.rs — 可见性图

构建节点间的可见性图（Visibility Graph），用于计算绕过障碍物的最短路径。部分路由算法（如 Spline）依赖此模块实现障碍物感知。

edge_geometry.rs — 几何计算

提供边路由通用的几何计算工具：线段交点、点到线段距离、贝塞尔曲线求值、路径简化等。

label_avoidance.rs — 标签避让

边标签（Edge Label）的智能定位。计算标签在边上的最佳放置位置，避免遮挡节点或其他标签。

parallel_edges.rs — 平行边处理

处理同一对节点之间的多条边（平行边 / 多重边）。通过偏移量、不同曲率等方式使平行边在视觉上可区分。

选择决策指南

✅ 追求清晰规整

选 Orthogonal（默认推荐；SVG 渲染自动圆角）

🎨 追求美观柔和

选 Bezier（bezier { tension }）或 Spline

🔄 环形布局

必须选 Circular，弧形边与圆形排列天然契合

🧱 节点密集需避障

选 Spline，基于可见性图的避障多段样条路由

⚡ 追求性能

选 Straight，最低复杂度，适合边数少的场景

📝 ER 图 / 简单关系

选 Straight，直线 + 平行边偏移即可

🔍 Orthogonal 路由深入

Orthogonal 是 Drawify 的默认推荐路由算法，也是复杂度最高、功能最完善的基础路由。 以下是其核心机制的详细说明。

通道/槽位系统（Channel / Slot）

在节点之间的空白区域划分水平通道和垂直通道。每条通道由多个槽位组成， 路由时为边分配不重叠的槽位，从而避免边与边、边与节点的重叠。

评分式路径选择

对候选路径进行评分，选择总惩罚最低的路径。评分维度包括：

  • 弯折惩罚 — 每个拐角增加惩罚，鼓励直线路径
  • 交叉惩罚 — 与已有边交叉增加惩罚
  • 绕行惩罚 — 路径长度超出曼哈顿距离越多，惩罚越大

路径简化

初步路由后，对折线进行简化：移除共线的冗余中间点、合并相邻的同方向段， 减少路径点数并提升渲染性能。
