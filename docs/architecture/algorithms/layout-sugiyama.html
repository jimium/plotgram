Sugiyama 分层布局算法

一套面向有向图 / DAG 的经典四阶段分层布局流水线。在 Drawify 中用于 流程图、架构图、状态机、活动图 等需要体现"方向性"与"层次关系"的可视化场景。

4-Phase Layered DAG Barycenter Crossing Min Iterative Coordinate

📌 算法概览

Sugiyama 算法（Kozo Sugiyama, 1981）将布局问题分解为 四步串行流水线。每一步专注一个美学目标，并把复杂问题降维成可快速求解的子问题。

适用场景

  • 流程图 Flowchart：步骤顺序明确、依赖单向
  • 架构图 Architecture：层 / 模块依赖、调用链
  • 状态机 State diagram：状态转移有向
  • 活动图 / 任务依赖图：有明确的"完成→进行"关系

不适用场景

  • 无向图、社交网络（建议力导向 force_directed）
  • 大量环（环需要被反转，会产生"绕回"边）
  • 语义分组优先、层次不明显（建议 mindmap 或力导向）

核心常量（来自 sugiyama.rs）

H_GAP = 60 px
V_GAP = 80 px
CROSSING_ITERATIONS = 6
CROSSING_WEIGHT = 1.0
BENDING_WEIGHT = 0.5
EDGE_LENGTH_WEIGHT = 0.3

ER 模式会使用真实节点尺寸与中心坐标来对齐网格，从而在 ER 图上得到更紧凑的布局。

🧩 四阶段流水线

P1

Cycle Removal · 环消除

使用 DFS 遍历，识别 back edge（在 DFS 树上指向祖先的边）， 然后在布局阶段把这些边反向，从而把原图变成 DAG。

  • 不删除边，只是在"分层/排序"阶段反向使用
  • 最终渲染时按原方向绘制，因此环会以"绕回"方式呈现

复杂度 O(V + E)，线性。

P2

Layer Assignment · 层分配

对 DAG 进行拓扑排序，按最长路径分配层：

  • 每个节点的 layer[v] = max(layer[u]) + 1，u 为前驱
  • 源节点 layer = 0（或最顶层）
  • 同一层的节点理论上没有"先后"语义，为 P3 留出自由度

复杂度 O(V + E)。垂直（纵向）布局时，layer 对应 y 坐标；水平布局时，layer 对应 x 坐标。

P3

Crossing Min + Bend Optimization · 交叉与弯折优化

使用 Barycenter（重心）启发式，在相邻层之间做 上扫 / 下扫（up/down sweeps）迭代排序。

  • 把每个节点排序到其邻居（上一层 / 下一层）的加权平均位置
  • 目标：最小化加权和 —— crossing × 1.0 + bending_var × 0.5
  • 弯折权重较低，允许少量平滑折线路径，以换取更少交叉

每 sweep O(E + L log L)；通常 5~20 次 sweep 后稳定。

P4

Coordinate Assignment · 坐标分配

在层内顺序已固定的前提下，再次 上下扫迭代， 把每个节点推向其父 / 子 barycenter，使得长边缩短，节点向"父-子重心"靠拢。

  • 目标项：edge_length × 0.3
  • 坐标优化后再 对齐网格（以 H_GAP / V_GAP 为步长）
  • 支持 左→右（水平） 与 上→下（垂直） 两种流向

最后计算 group bounds 与 padding，输出中心坐标。

💡 Tips：什么时候选择 Sugiyama

✅ 当你需要"阅读顺序"

  • 读者可以顺着 layer 顺序自上而下/自左向右理解流程
  • 同层=同级语义，适合步骤 / 阶段可视化

⚠️ 当图里有环时

  • P1 会把环"绕回"以虚线或反向路径呈现
  • 环越多，布局越"绕"；此时考虑 circular 或 force_directed

🎯 当你想控制流向

  • 支持 top→bottom 与 left→right
  • 结合 ER-mode 可让节点以真实尺寸对齐到规则网格
