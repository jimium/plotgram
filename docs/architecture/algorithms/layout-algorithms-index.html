节点布局算法

决定图中节点位置的算法，是图可视化的核心。选择合适的布局算法直接决定图形的可读性与美观度。

Sugiyama 分层布局

Sugiyama Hierarchical Layout

经典 DAG 分层布局，通过层级分配、交叉最小化与坐标优化，生成清晰的上下/左右流向图。

确定性 ✓ 规模 ≤ 300 节点 适用 DAG / 流程图
查看详情

Fruchterman-Reingold 力导向

Force-Directed (FR) Layout

基于 Fruchterman-Reingold 引斥力模拟的力导向布局，增强分组语义，适合架构图与微服务拓扑。

确定性 ✗ 规模 ≤ 180 节点 适用 架构图 / 微服务拓扑
查看详情

Mindmap 思维导图布局

Mindmap Layout

面向树形结构的专用布局，支持径向/自上而下/自左向右三种展开方向，根/主干/分支/叶节点差异化尺寸。

确定性 ✓ 规模 ≤ 200 节点 适用 思维导图 / 知识树
查看详情

Circular 圆形布局

Circular Layout

节点排列在圆周上，支持单环和多环模式（基于双连通分量分解），使环结构和闭环关系直观可见。

确定性 ✓ 规模 ≤ 120 节点 适用 状态图 / 闭环关系
查看详情

Sequence 时序图布局

Sequence Layout

专为时序图设计：参与者水平展开，消息按声明顺序垂直排列，内置边几何生成。

确定性 ✓ 规模 ≤ 200 消息 适用 时序图
查看详情

Sugiyama V2 增强分层布局

Sugiyama V2 Hierarchical Layout

Greedy 去环 + NS-style 紧边压缩 + 加权中位数排序 + BK-style 坐标分配，更紧凑的层级与更均匀的横向分布。

确定性 ✓ 规模 ≤ 320 节点 适用 复杂 DAG / 流程编排
查看详情

Architecture V2 架构图布局

Architecture V2 Layout

专为架构图设计的混合布局：宏观层将分组视为超级节点做拓扑分层，微观层对组内节点做 Sugiyama 分层。

确定性 ✓ 规模 ≤ 250 节点 适用 架构图 / 微服务拓扑

边路由算法

决定节点之间连线路径的算法。好的路由算法能减少边交叉、避免遮挡，显著提升图形可读性。

边路由算法集

Straight / Orthogonal / Bezier / Circular / Spline

5 种路由策略：直线、正交折线、贝塞尔曲线、环形弧线、避障样条，满足不同视觉风格与可读性需求。

确定性 ✓ 规模 任意 适用 全场景覆盖

推荐选型

根据业务场景选择最佳布局 + 路由组合，快速定位适合你需求的算法方案。

业务场景 布局算法 路由算法 说明
流程图 / 审批流 Sugiyama Orthogonal 经典分层 + 正交折线，流程步骤清晰可读
思维导图 / 知识树 Mindmap Bezier 中心发散 + 贝塞尔曲线，自然贴合导图风格
状态机 / 闭环关系 Circular Circular 环形排列 + 弧线连接，状态流转直观展示
架构图 / 分组组件图 Architecture V2 Orthogonal 分组感知分层 + 正交折线，组件边界清晰
网络拓扑 / 自由依赖图 Force-Directed Orthogonal FR 力导向分散 + 正交折线，高密度网络清晰可辨
时序交互 Sequence 内置路由 时序图自带消息线路由，无需额外选择
ER 关系图 Sugiyama Straight 分层布局 + 直线连接，实体关系简洁明了
高密度流程 DAG Sugiyama V2 Orthogonal 优化版分层 + 正交折线，大规模 DAG 性能与可读性兼得
