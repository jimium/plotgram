Drawify 布局算法分类

节点布局与边路由的分层类型 · 分类体系 · 图类型适用矩阵
版本参考：crates/drawify-core/src/layout · 生成用途：黑白打印

一句话结论

「分层类型」指 Sugiyama 流水线（去环 → rank 层分配 → 层内排序 → 坐标分配）： 共 5 个算法名 — sugiyama、sugiyama-v2、flowchart、er、architecture。 mindmap / sequence 有层级阅读感但算法不同； force-directed / circular / state 不属于 Sugiyama 分层。 项目用四套并行标准分类，没有单一的「分层 / 非分层」枚举。

1. 两个独立维度

布局系统分为节点布局与边路由两层，职责不同，注册名也不同。 orthogonal 是边路由算法，不是节点布局算法。

维度 职责 注册名示例
节点布局
LayoutStrategy		 决定节点与 group 的几何位置 sugiyama-v2、architecture、force-directed …
边路由
EdgeRoutingStrategy		 节点定好后，计算边的路径与标签位置 orthogonal、straight、bezier、circular …

2. 哪些是「分层类型」

项目中的「分层」主要指 Sugiyama 分层布局：将有向边沿统一主方向（自上而下或自左而右）排列， 使控制流 / 依赖方向可读。

2.1 Sugiyama 分层家族（明确分层）

算法名 本质 适用图类型 备注
sugiyama-v2 通用 Sugiyama 引擎（NS 层分配 + 加权中位数排序 + BK 坐标） flowchart、state、er、custom 高级通用选项
flowchart 门面 → sugiyama-v2 + FLOWCHART_PRESET flowchart、custom 流程图默认
er 门面 → sugiyama-v2 + ER_PRESET er ER 图默认
sugiyama 旧版 Sugiyama（backup/，A/B 对照） flowchart、er、custom 保留旧实现
architecture 混合分层：组间宏观 Sugiyama + 组内 Sugiyama / layout hint architecture、custom 架构图默认

2.2 有层级感，但不是 Sugiyama 分层

算法 层级方式 说明
mindmap 树深度 根节点向外辐射或 TB/LR 展开；假设树 / 近树拓扑，不做 DAG rank 分配
sequence 固定语义网格 参与者水平一行，消息按声明顺序纵向堆叠；布局阶段内置边几何

2.3 非分层

算法 范式 说明
force-directed 力导向 库仑斥力 + 弹簧引力模拟；无固定层；非完全确定性
circular 圆周排列 节点排列在圆周上；适合环结构 / 强连通分量
state 圆周排列（门面） 底层共享 circular 引擎；状态图默认

3. 分类体系（四套并行标准）

代码中没有 LayoutKind::Layered 之类单一枚举，而是通过以下标准组合描述算法。

3.1 按几何范式

节点布局（LayoutStrategy） ├── 分层（Sugiyama 系） sugiyama, sugiyama-v2, flowchart, er, architecture ├── 树形 / 辐射 mindmap ├── 圆周 circular, state ├── 力导向 force-directed └── 语义约束 sequence

3.2 按图拓扑特征（选型依据）

拓扑特征 推荐布局 原因
有向无环图（DAG） sugiyama-v2 分层布局天然适配 DAG，层级方向与边方向一致
有环有向图 sugiyama-v2 去环阶段处理环，分层后环变为跨层回边
树 / 近似树 mindmap 专用树布局，根节点突出，子树紧凑
强连通 / 环结构 circular 圆周排列使环结构直观可见
网状 / 弱层级 force-directed 力导向自然散开，不强制分层
分组语义强 architecture 两层 Sugiyama：组间宏观 + 组内紧凑
时序语义 sequence 强语义专用布局（参与者 + 时间步）

3.3 按图类型适用关系

每个算法通过 LayoutStrategy::applicable_diagram_types() 与可选的 supports_custom() 声明适用范围；校验与 Playground catalog 均走此路径。

图类型 默认布局 可选布局 默认边路由
flowchart flowchart sugiyama-v2、sugiyama、force-directed orthogonal
architecture architecture force-directed orthogonal
state state circular、sugiyama-v2 circular
er er sugiyama-v2、sugiyama、circular straight
sequence sequence （仅自身） 内置，无 edge_routing
mindmap mindmap （仅自身） bezier

3.4 按实现结构（引擎 vs 门面）

底层引擎 门面 / 直连 说明
sugiyama_v2 flowchart、er、sugiyama-v2 不同 preset：FLOWCHART / ER / GENERIC
circular state、circular state 为图类型默认门面
architecture_v2 architecture 含 group 时两阶段布局；无力导向混合
独立实现 mindmap、sequence、force-directed 无共享引擎
backup/sugiyama sugiyama 旧版保留，便于 A/B
