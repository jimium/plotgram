SVG 内部按钮在 <img> 中能工作吗？

同一个带 SMIL 按钮的 SVG，用四种方式嵌入

答案：不能。 <img> 将 SVG 视为静态光栅图像，浏览器对其施加严格的安全隔离：
  • ❌ SMIL 的 begin="btn.click" 不触发——点击事件不会穿透到 SVG 文档内部
  • ❌ CSS :hover 伪类不生效——鼠标悬停在 img 上，SVG 内部元素感知不到
  • ❌ SVG 内的 <a xlink:href> 链接不可点击
  • ❌ SVG 内的 <script> 不执行
但在 <object>/<iframe>/直接打开 SVG/内联 SVG 中按钮完全正常。

❌ <img src> 按钮不工作

<img src="btn-in-svg.svg">

按钮点不动、hover 不变色、链接打不开。SVG 被当作静态图片。循环动画生效，但交互完全失效。

❌ SMIL 按钮点击 · ❌ CSS :hover · ❌ 链接点击 · ✅ CSS 动画(循环)

✅ 直接打开 .svg 按钮正常

点击在新窗口打开 btn-in-svg.svg →

SVG 作为独立文档加载。按钮点击、hover 变色、链接跳转全部正常。SMIL begin="click" 正常工作。

✅ SMIL 按钮点击 · ✅ CSS :hover · ✅ 链接点击 · ✅ 动画

✅ <object data> 按钮正常

<object data="btn-in-svg.svg">

SVG 作为独立插件文档加载。交互完全正常，可以点击按钮、hover、点链接。

✅ SMIL 按钮点击 · ✅ CSS :hover · ✅ 链接点击 · ⚠️ 同域 JS 可控

✅ <iframe src> 按钮正常

<iframe src="btn-in-svg.svg">

SVG 在独立 frame 中加载。交互完全正常，有沙箱隔离。

✅ SMIL 按钮点击 · ✅ CSS :hover · ✅ 链接点击 · ⚠️ 同域 JS 可控

对 Drawify 的设计影响：
1. 纯 SVG 导出（无交互版）：适合 <img> 嵌入 GitHub README、文档——只放循环动画（边数据流）和一次性入场动画，不放按钮。用户双击打开 .svg 时按钮可用，但通过 img 嵌入时按钮不工作也不影响（因为无按钮场景不需要交互）。
2. Steps 分步演示：需要按钮控制，所以不能用纯 SVG + <img> 嵌入。有两个选择：
  • 用 HTML 导出（内联 SVG）：按钮放在 HTML 中（HTML <button>），JS 控制内联 SVG——最灵活，推荐为主力格式
  • 用 纯 SVG 导出但要求用 <object>/<iframe> 或直接打开：按钮画在 SVG 内部（SVG <rect> + SMIL 或 CSS :target），适用于 Notion/Confluence 支持 object 的场景
3. GitHub README 嵌入：只能用 <img>，所以只能展示自动播放的循环动画（数据流、脉冲等），不展示 Steps 交互——这是可接受的，GitHub 里的 README 本来就不需要交互。
