SVG 内嵌 CSS 动画简明教程

CSS 写在 SVG 的 <defs><style> 里，文件自包含。 用 <img> 嵌入、双击打开、或内联到 HTML，内部动画都能播放。 详见 嵌入方式对比。

1基本写法

动画样式放在 SVG 文档内部，与图形结构同文件。推荐用 CDATA 包裹，避免 < 被 XML 解析器误读。 

<!-- SVG 文件结构 -->
<svg>
  <defs>
    <style><![CDATA[
      @keyframes fade { from { opacity: 0; } to { opacity: 1; } }
      .box { animation: fade 1.2s ease-out infinite alternate; }
    ]]></style>
  </defs>
  <rect class="box" .../>
</svg>

2transition 与 @keyframes

transition：属性从 A 平滑变到 B，通常由 class 切换或 :hover 触发，适合 Patch 后「滑到新位置」。
@keyframes：定义多关键帧时间轴，适合入场、循环、绘制线条。 

/* transition：两个状态之间插值 */
.node { transition: transform 0.5s ease; }
.node.moved { transform: translate(90px, 0); }

/* @keyframes：按时间百分比定义 */
@keyframes pop {
  0%   { opacity: 0; transform: scale(0); }
  100% { opacity: 1; transform: scale(1); }
}

3transform：位移、缩放

节点动画优先用 transform 和 opacity（GPU 合成，性能好）。 SVG 元素默认按画布坐标变换，加 transform-box: fill-box + transform-origin: center 可让缩放绕图形中心。

4边的绘制与数据流动

stroke-dasharray + stroke-dashoffset 是 SVG 边动画的核心技巧。 绘制效果：从大 offset 动画到 0；流动效果：循环改变 offset，表达调用方向。 

.edge-draw {
  stroke-dasharray: 180;
  stroke-dashoffset: 180;
  animation: draw 1.2s ease-out forwards;
}
@keyframes draw { to { stroke-dashoffset: 0; } }

5animation-delay：交错入场

多个节点依次出现：同一 @keyframes，不同 animation-delay。 Drawify Patch 动画中，Add 节点常用此模式表达「逐个落地」。

6淡入淡出：增删与路径变更

节点 Add → opacity 0→1；Remove → 1→0。 边路径点数变化时，不做 d 属性插值（点对不齐会失败），用旧边淡出 + 新边淡入交叉渐变。

7:hover 交互（非 <img> 嵌入时）

transition 配合 :hover 可做悬浮高亮。 注意：用 <img src="x.svg"> 嵌入时，指针事件进不去，hover 不生效；直接打开 SVG 或用 <object> / 内联 SVG 可以。

8能力边界与性能

做架构图动画时的实用对照表。

能力CSS 方案说明
节点入场 / 出场 opacity + transform 推荐，性能好
位置变更（Patch） transition: transform 初始 translate + 过渡到目标
边绘制 / 数据流 stroke-dashoffset 架构图标配
边路径变形 交叉淡入淡出 勿动画 d 属性
颜色渐变 transition: fill 可用，但触发重绘
分步 Steps 切换 外部 JS 改 class 纯 SVG 不行，需 HTML 播放器
无障碍 prefers-reduced-motion 一行媒体查询关闭动画 
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}
