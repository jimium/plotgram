三种按钮 × SVG 协作模式

模式 A：无按钮，自动播放

纯 SVG · 零 JS

Patch 动画、参数化指令、循环动画。打开 SVG 即播放，不需要用户操作。 CSS animation-delay 做 stagger，SMIL/CSS 自动执行。

✅ 单文件 SVG，双击打开就有动画
✅ 适合 Patch 过渡、入场动画、边数据流
❌ 无法控制暂停/播放

模式 B：按钮画在 SVG 里

纯 SVG · 零外部 JS

按钮本身也是 SVG 元素（<rect>+<text>）， 用 SMIL begin="btn.click" 或 CSS :target 控制。 单个 .svg 文件即可交互。

✅ 仍是单文件 SVG，GitHub 可嵌入
✅ 按钮是 SVG <rect>，SMIL 监听 begin="id.click"
❌ 控制能力有限（不能做复杂逻辑），按钮样式简单

模式 C：HTML 按钮 + JS 控制

HTML 导出 · 最灵活

按钮是 HTML <button>，在 SVG 外面。 JS 通过 document.getElementById 操作 SVG 内部元素， 添加/移除 CSS class 来触发动画。

✅ 按钮样式丰富，可做时间轴、进度条等复杂 UI
✅ JS 可实现任意逻辑（自动播放、跳转、循环）
❌ 需要 HTML 文件（SVG 单独文件无按钮）

三种模式与导出格式的对应关系

导出格式 按钮模式 协作方式 适用场景
.svg（Patch/参数化） 模式 A：无按钮 CSS/SMIL 自动播放 Agent 微调过渡、高亮截图
.svg（Steps 纯 SVG 版） 模式 B：SVG 内按钮 SMIL begin="id.click" GitHub/Notion 嵌入（只接受 SVG）
.html（Steps 完整版） 模式 C：HTML 按钮 JS DOM 操作 class/属性 演示、培训、架构讲解（浏览器打开）
